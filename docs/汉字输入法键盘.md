这是一个极其敏锐且直击物理交互本质的工程问题。

在传统的认知中，标准键盘有 104 个按键。我们要把它们全部塞进只有 **36 键** 的 Han-Board 里，看似是一个不可能的物理悖论。但这正是 104 键 QWERTY 键盘落后于时代的证明：**为了增加功能，它只能极其愚蠢地在二维平面上不断“摊大饼”，导致程序员每天的手腕位移距离高达数百米。**

在我们的中文原生计算架构中，Han-Board 引入了现代极客键盘（如 QMK/ZMK 固件驱动的 36 键分体键盘）的终极空间魔法：**“时空复用（Mod-Tap）”** 与 **“多维空间折叠（Layers）”**。

这套协议被定义为 **《RFC-ZH003: Han-Board 空间折叠与全域映射白皮书》**。核心物理法则只有一条：**手掌的掌根必须死死焊在桌面上，一毫米都不准平移。**

### 法则一：时空复用（Home Row Mods）—— 彻底消灭“程序员小指痛”

标准键盘最反人类的设计，就是把 `Ctrl`、`Shift`、`Alt` 和 `Win/Mac` 键放在最边缘，逼迫最脆弱的小拇指进行极其扭曲的极限拉伸（即著名的 Emacs Pinky 综合征）。

在 Han-Board 上，**所有的系统修饰键都不再占用独立按键，而是隐身于你的“基准键位（Home Row，即双手自然悬停的中排 8 个键）”之下。**

底层系统通过**按压时长**（通常以 150 毫秒为界）来判定你的意图：

* **短敲（Tap）**：正常输出汉字部首 / 极码指令。
* **长按（Hold）**：按键瞬间化身为修饰键！

**【左右手中排（休止位）多态映射表】**

* **食指** 长按 = `Shift`
* **中指** 长按 = `Ctrl`
* **无名指** 长按 = `Alt`
* **小指** 长按 = `Win / GUI`

> 💥 **降维打击演示：执行 `Ctrl + C`（复制）**
> 在传统键盘上，你需要手腕左移，左手小指别扭地按 Ctrl，食指找 C。
> **在 Han-Board 上：你的左手中指轻轻压住不放（瞬间激活 Ctrl），右手食指同时轻敲对应 C 的位置。手腕位移距离：0 毫米。极度优雅。**

### 法则二：大拇指的“维度引擎”与四大拓扑层

36 键分出了 **左 3 + 右 3 的超级拇指簇**。大拇指是人类力量最大、最灵活的手指，在传统键盘上却只用来敲一个毫无信息量的空格。
在 Han-Board 中，大拇指是穿梭物理空间的**“离合踏板（Clutch）”**。只要大拇指踩下不同的键，主键盘区的 24 个按键就会瞬间“跃迁”到不同的功能层（Layer）。

#### 🌌 维态 0：太极态（Base Layer / 汉字基底）

* **状态**：大拇指未长按任何维度键。
* **功能**：运行 `HuaCode` 二维和弦算法。双手齐下，单次物理击键生成一个汉字或词组。这是键盘的日常输出态。

---

#### 🔢 维态 1：算筹态（Num & Math Layer）

人类大脑处理数字最直觉的物理模型，是 3x3 的九宫格（计算器/小键盘），而不是主键盘上反人类的一字长蛇阵（1~0）。

* **触发条件**：**左手大拇指按住 `[算筹踏板]` 不放。**
* **空间坍缩**：此时，**右手的 12 个键瞬间变成完美的九宫格数字小键盘**。左手变成数学符号与编程括号。

  【左手：逻辑符与界定符】               【右手：绝对正交数字小键盘】
 [ + ]  [ - ]  [ * ]  [ / ]          [ = ]  [ 7 ]  [ 8 ]  [ 9 ]

> 💥 **降维打击演示：输入 IP 地址 `192.168.1.1**`
> 左拇指压住，右手完全不需要像传统键盘那样离开主区长途跋涉到最右侧，而是直接在原地盲打九宫格。打完松开左拇指，瞬间切回中文状态。

#### 🧭 维态 2：司南态（Nav & Edit Layer）

修改文本或写代码时，高频移动光标是最大的痛点。

* **触发条件**：**右手大拇指按住 `[司南踏板]` 不放。**
* **空间坍缩**：右半区变成光标导航（类似 Vim 的倒 T 型结构），左半区变成高频编辑指令。

【Layer 2：司南态】(右拇指 Hold 激活)

> 💥 **降维打击演示：选中当前行并删除**
> 左手食指长按（激活 `Shift`），右手大拇指压住（切入 `司南态`），右手中指敲击 `Home`，再敲 `End`（选中整行），最后右拇指松开，轻敲 `退格键`。
> **全程没有任何悬腕找方向键和鼠标的动作，双手犹如弹奏钢琴。**

### 🚀 终极实战检验：极其恐怖的连招（Combo）

让我们模拟一个真实的极客工作流。
**任务：用 `DaoLang (极码)` 写一段代码，输入变量后，发现拼错了，光标回退修改，然后运行（F5）。**

* **传统 104 键盘的灾难：**
打拼音 -> 选词 -> 手伸到顶部打数字 -> 手腕挪到右下角找方向键 -> 移回主区按 Backspace -> 移回右下角方向键回正 -> 视线寻找顶部的 F5 键。
*(肌肉横跳至少 7 次，耗时 3~4 秒)*
* **Han-Board 36 键的“神经微动”：**
1. **打字**：双手和弦两下，瞬间上屏 `客验令` (UserToken)。
2. **打数字**：左拇指压住（切算筹态），右手原地极速敲出 `1 0 2 4`。
3. **发现打错了想改**：松开左拇指，右拇指压住（切司南态），右手食指敲三下（←←←），退到 `0` 后面。
4. **修改**：松开右拇指敲击一次（Backspace 删掉 0），左拇指压住（切算筹态），右手轻敲 `2`。
5. **运行**：双拇指同时压住（切天机态），左手食指敲击 F5 位置。
*(双手手腕死死锁定在原位 0 毫米平移，纯靠神经反射的指尖微操，耗时 1 秒内完成)*
